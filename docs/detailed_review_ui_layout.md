# Detailed Flagged Review View UI/UX Layout - Conceptual Design\n\n## Overview\n\nThis page will display comprehensive information for a single flagged review, enabling Trust & Safety Analysts to investigate further. It will be accessible by clicking on a flagged review entry from the dashboard or a dedicated flagged reviews list.\n\n## Key Sections and Components:\n\n1.  **Header/Navigation:**\n    *   Back button/link to the dashboard or flagged reviews list.\n    *   Review ID for easy reference.\n\n2.  **Review Summary Card (Top Section):**\n    *   **Flagged Review ID:** Unique ID for the flag instance.\n    *   **Review ID:** The ID of the original review.\n    *   **Product ASIN:** Identifier for the product the review is about.\n    *   **Reviewer ID:** Identifier for the user who wrote the review.\n    *   **Date Flagged:** Timestamp of when the review was flagged by the system.\n    *   **Flagging Reason:** The primary reason the review was flagged (e.g., \"Keyword Match: 'received for free'\").\n    *   **Current Status:** (e.g., PENDING\_REVIEW, APPROVED, DISMISSED).\n\n3.  **Original Review Content (Middle Section):**\n    *   **Review Content:** The full text of the original review.\n    *   **Original Review Date:** Date when the review was originally posted.\n    *   **Review Rating:** The star rating given by the reviewer.\n\n4.  **Analyst Actions & Notes (Bottom Section):**\n    *   **Action Buttons:**\n        *   \"Approve Flag\": Marks the flag as valid, potentially triggering further actions (e.g., review removal, reviewer action).\n        *   \"Dismiss Flag\": Marks the flag as invalid (false positive).\n    *   **Analyst Notes:** A text area for analysts to add their findings, reasons for approval/dismissal, or any relevant comments.\n    *   **Resolved By / Resolved At:** Displays which analyst resolved the flag and when.\n\n## Wireframe Sketch (Text-based
